What Are Prompt Frameworks?

How Frameworks Enhance Your Simulation Prompt

A prompt framework is a structured instructional approach embedded directly into your Claude prompt. It tells Claude not just what to build, but how students should cognitively engage with it.

Think of it this way: Without a framework, Claude builds a simulation. With Chain of Thought, Claude builds a simulation where students must verbalise their reasoning before every decision. With Tree of Thought, students evaluate three possible futures before the real one is revealed.
All 9 Frameworks

Framework Reference — What Each One Does

RTF
Relevant · Transparent · Feedback
Grounds the simulation in student prior knowledge, makes learning objectives explicitly visible at the start, and builds structured feedback loops into every round outcome.
SPARK
Specific · Purposeful · Adaptive · Relevant · Knowledge-building
Ensures every simulation element has a declared purpose and that difficulty adapts to how students are performing.
CREATE
Creative · Relevant · Engaging · Authentic · Thoughtful · Exploratory
Opens the simulation beyond preset scenarios — students can propose their own variable values and design their own "what if" extension.
GRADE
Goals · Rigor · Audience · Differentiation · Evidence
Assessment-focused framework. Aligns each round to a specific IB criterion, provides three differentiated entry points, and generates evidence students can submit.
Persona Prompting
Role-based Expert Framing
Claude adopts the persona of a domain expert throughout the simulation.
Chain of Thought (CoT)
Step-by-Step Reasoning Externalisation
Before advancing each round, students must complete a reasoning chain in writing.
Tree of Thought (ToT)
Multi-Path Futures Evaluation
Before the real result is revealed, three plausible future scenarios are shown based on the student's decisions.
ReAct
Reason · Act · Observe · Reflect
Structures every round as a four-stage explicit cycle.
Self-Consistency (SC)
Multi-Perspective Verification
After the student makes a prediction, three advisor characters each make their own prediction with brief reasoning.

What & Why

What Are SVG Simulations?

A single HTML file that runs in any browser — no login, no app, no WiFi needed after download. Students interact with sliders, make predictions, and watch an animated visual world respond to their choices in real time.

Works for every IB subject Any topic with interacting variables that change over time can be simulated.

// SVG CLASSROOM SIMULATION — MASTER PROMPT You are an [SUBJECT] teacher creating an interactive simulation for Grade [GRADE] students. SIMULATION TOPIC: [topic] | CURRICULUM: [IB MYP/DP subject] DELIVER AS: single self-contained HTML file — animated SVG, resource bars, trend chart, prediction widget, IB reflection panel, splash screen, reset button, offline capable, responsive.
Before Class

Quality Checklist

  • Open HTML in Chrome or Firefox
  • Advance through all rounds — check no JavaScript errors (F12 → Console tab)
  • Verify prediction widget disables Advance button until prediction is selected
  • Check resource bars animate and shift colour green → amber → red correctly
  • Test Reset button restores all starting values
  • Disconnect WiFi and reload — should work offline

Click New. Name it e.g. ib-eco-sim. Tick "Add a README file" and click Create repository.

Click Add file → Upload files. Name it index.html. Click Commit changes.

Go to Settings → Pages. Under Source, select Deploy from a branch, choose main and / (root). Click Save.

Copy the URL from Settings → Pages. Paste into ManageBac or Google Classroom.

Assessment Security

Keeping Assessment Simulations Private

The problem: If a student follows your GitHub profile, they can see all public repositories.
A

Unpublish Pages (simplest)

Keep repo Public but unpublish Pages until assessment day. Go to Settings → Pages → Source → None. Re-enable on assessment day.

B

Entering code to view the assessment

Add a code-gate page — students can't view without entering the code first.

C

Separate assessment account (recommended)

Create a second GitHub account (e.g. fountainhead-assessments) that students don't follow.

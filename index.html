What & Why

      
What Are SVG Simulations?

      
A single HTML file that runs in any browser — no login, no app, no WiFi needed after download. Students interact with sliders, make predictions, and watch an animated visual world respond to their choices in real time.

      

        🎨 SVG Scene
        +
        ⚙️ Decision Sliders
        +
        📊 Resource Bars
        +
        🔮 Predict → Reveal

Replace every [BRACKET] with your own content. Use the Simulation Prompter tab to do this automatically.

      

        
// SVG CLASSROOM SIMULATION — MASTER PROMPT
// Replace every [BRACKET] then paste into Claude

You are an [SUBJECT] teacher creating an interactive simulation for Grade [GRADE] students.

SIMULATION TOPIC: [e.g. "population growth and resource scarcity"]
CURRICULUM: [e.g. "IB MYP Individuals & Societies"]
KEY CONCEPT: [e.g. "Sustainability"]
RELATED CONCEPTS: [e.g. "Scarcity, Interdependence"]
LEARNER PROFILE: [e.g. "Thinkers, Caring"]
ATL SKILLS: [e.g. "Thinking, Research"]
SETTING: [Describe the visual scene]
SLIDERS: [3-6 student-controlled variables]
METRICS: [4-6 tracked resources, 0-100]
ROUNDS: [number]
VISUAL STYLE: [describe look and feel]

DELIVER AS: a single self-contained HTML file with animated SVG,
resource bars, prediction widget, trend chart, reflection panel.

AFTER GENERATING, ask the teacher:
"Would you like: A) Simplified version B) HL Extension
C) Retheme D) Teacher Panel E) Student Worksheet F) Bilingual mode"

    


    
    

      
Power Tips

      
Getting Better Results

      

        

          
🎯 Be specific about visuals

          
Don't say "make it interactive." Say "when forest drops below 30%, trees shrink and turn brown."

        

        

          
⚖️ Build trade-offs

          
Best simulations have decisions that improve one metric while hurting another.

        

        

          
🔄 Iterate in same chat

          
After Claude generates, say "change the colour scheme" — it regenerates keeping everything else.

        

      

    


    
    

      
Before Class

      
Quality Checklist

      
    
        
  • ☐ Open HTML in Chrome or Firefox and advance through all rounds
    • 
        
  • ☐ Verify prediction widget disables Advance until selected
    • 
        
  • ☐ Check resource bars animate green → amber → red
    • 
        
  • ☐ Confirm the visual scene changes when sliders are at extremes
    • 
        
  • ☐ Disconnect WiFi and reload — should work offline
    • 
        
  • ☐ Test on both projector resolution and tablet
